# Credit Scoring with Machine Learning

## Описание задачи
Бинарная классификация заёмщиков на основе их кредитной истории и социально-демографических характеристик. Целевая переменная `SeriousDlqin2yrs` принимает значение **1**, если заёмщик допустил просрочку платежа на 90 и более дней в течение двух лет, и **0** в противном случае.

## Данные
- **Источник:** Kaggle — [Give Me Some Credit Dataset](https://www.kaggle.com/datasets/brycecf/give-me-some-credit-dataset).
- **Размер:** 150 000 записей, 11 признаков.
- **Категориальные признаки:** отсутствуют — все признаки числовые.
- **Дисбаланс классов:** 6,68% дефолтов, 93,32% не-дефолтов.

## Статус проекта (текущий)
- ✅ **Выполнено:** полный разведочный анализ (EDA) — выявлены пропуски, выбросы, корреляции, дисбаланс. Результаты оформлены в `notebooks/01_eda.ipynb`.
- ⏳ **В процессе:** подготовка к предобработке данных (заполнение пропусков, обработка выбросов, масштабирование).
- 📅 **Планируется:** обучение и сравнение моделей, балансировка классов, подбор гиперпараметров.

### Ключевые выводы из EDA
- **Пропуски:** только в признаках `MonthlyIncome` (19,8%) и `NumberOfDependents` (2,6%). Решено заполнять: `MonthlyIncome` — медианой, `NumberOfDependents` — модой (0).
- **Выбросы:** аномальные значения в возрасте (0 лет), количестве просрочек (до 98), коэффициенте долга (до 329 664), доходе (до 3 млн). Будут обработаны (винсоризация или оставлены для устойчивых моделей).
- **Корреляции:** сильная положительная связь (0,98–0,99) между признаками, описывающими просрочки разной длительности. Это дублирование информации; для линейных моделей может создавать мультиколлинеарность.
- **Связь с целевой переменной:** слабая (максимальная корреляция 0,13 у `NumberOfTime30-59DaysPastDueNotWorse`).

## Планируемые методы
- **Предобработка:**
  - Заполнение пропусков (медиана / мода).
  - Обработка выбросов (винсоризация или преобразование).
  - Масштабирование числовых признаков (StandardScaler — для логистической регрессии).
  - Балансировка классов (опционально: SMOTE или class_weight).
- **Модели:**
  - Логистическая регрессия (baseline).
  - Случайный лес (Random Forest).
  - XGBoost.
  - CatBoost.
- **Оценка:** Precision, Recall, F1-мера, ROC-AUC.

## Как запустить (локально)
1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/dmitrii-a-piskunov/credit-scoring-ml.git
   cd credit-scoring-ml
   ```
2. Установить зависимости (рекомендуется виртуальное окружение):
   ```bash
   pip install -r requirements.txt
   ```
3. Запустить Jupyter Notebook:

   jupyter notebook notebooks/01_eda.ipynb
