# Credit Scoring with Machine Learning

## Описание задачи
Бинарная классификация заёмщиков на основе их кредитной истории и социально-демографических характеристик. Целевая переменная `SeriousDlqin2yrs` принимает значение **1**, если заёмщик допустил просрочку платежа на 90 и более дней в течение двух лет, и **0** в противном случае.

## Данные
- **Источник:** Kaggle — [Give Me Some Credit Dataset](https://www.kaggle.com/datasets/brycecf/give-me-some-credit-dataset).
- **Размер:** 150 000 записей, 11 признаков.
- **Категориальные признаки:** отсутствуют — все признаки числовые.
- **Дисбаланс классов:** 6,68% дефолтов, 93,32% не-дефолтов.

## Статус проекта (текущий)
- ✅ **Выполнено:** полный разведочный анализ (EDA) — выявлены пропуски, выбросы, корреляции, дисбаланс.
- ✅ **Выполнено:** предобработка данных — заполнение пропусков (медиана/мода), удаление выбросов (возраст, просрочки >30), масштабирование (StandardScaler), разделение на train/val (80/20), сохранение данных и scaler.
- ✅ **Выполнено:** обучение и сравнение моделей: логистическая регрессия (baseline), случайный лес, XGBoost, CatBoost.
- 📅 **Планируется:** финальное тестирование лучшей модели (XGBoost) на `cs-test.csv`, эксперимент с балансировкой классов (SMOTE), интерпретация результатов с помощью SHAP.

### Ключевые выводы из EDA
- **Пропуски:** только в признаках `MonthlyIncome` (19,8%) и `NumberOfDependents` (2,6%). Заполнены: `MonthlyIncome` — медианой (5400), `NumberOfDependents` — модой (0).
- **Выбросы:** аномальные значения в возрасте (0 лет), количестве просрочек (до 98), коэффициенте долга (до 329 664), доходе (до 3 млн). Удалены строки с `age` 0 или >100, а также с просрочками >30 (0,19% данных). Остальные выбросы оставлены, так как ансамблевые модели устойчивы к ним.
- **Корреляции:** сильная положительная связь (0,98–0,99) между признаками просрочек (дублирование информации). Для линейных моделей это создаёт мультиколлинеарность, но для ансамблей не критично.
- **Связь с целевой переменной:** слабая (максимальная корреляция 0,13 у `NumberOfTime30-59DaysPastDueNotWorse`).

### Результаты предобработки
- Заполнение пропусков (медиана / мода).
- Удаление явных выбросов (возраст, просрочки >30).
- Масштабирование числовых признаков (`StandardScaler`) — для логистической регрессии; сохранён объект `scaler.pkl`.
- Разделение на обучающую (80%) и валидационную (20%) выборки с сохранением пропорции классов (стратификация). Размеры: 119 773 / 29 944 строки.
- Данные сохранены в `data/processed/` (формат `.npy`).

### Результаты моделей (порог 0,5)
| Модель | Precision | Recall | F1-score | ROC-AUC |
|--------|-----------|--------|----------|---------|
| Logistic Regression | 0.256 | 0.619 | 0.363 | 0.817 |
| Random Forest | 0.223 | 0.710 | 0.340 | 0.858 |
| XGBoost | 0.213 | 0.771 | 0.334 | 0.865 |
| CatBoost | 0.213 | 0.769 | 0.333 | 0.865 |

## Дальнейшие планы
- Финальное тестирование всех обученных моделей (логистическая регрессия, случайный лес, XGBoost, CatBoost) на отдельном наборе данных `cs-test.csv` для оценки их обобщающей способности.
- Эксперимент с балансировкой классов (SMOTE) и сравнение качества с моделями, использующими только взвешивание классов.
- Интерпретация результатов лучшей модели с помощью SHAP (глобальная важность признаков и объяснение отдельных предсказаний).

## Как запустить (локально)
1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/dmitrii-a-piskunov/credit-scoring-ml.git
   cd credit-scoring-ml
   ```
2. Установить зависимости (рекомендуется виртуальное окружение):
   ```bash
   pip install -r requirements.txt
   ```
3. Запустить Jupyter Notebook:
   ```bash
   jupyter notebook
   ```
